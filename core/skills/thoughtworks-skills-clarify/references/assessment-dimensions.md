# 后端层级评估维度

对 `workflow.yaml` 中定义的每个层逐一判断是否需要开发。任一维度为"是"，则该层需要开发。

## Domain 层
- 是否引入新的业务概念（聚合根、实体、值对象）？
- 是否有新的业务规则或不变量？
- 是否需要新的仓储接口、领域事件、防腐层接口、领域服务？

## Infrastructure 层
- 是否需要新的数据库表？
- 是否需要实现新的 Repository 接口？
- 是否需要新的外部系统集成或中间件配置？

## Application 层
- 是否有新的业务用例需要编排？
- 是否需要协调多个聚合根或领域服务？
- 是否需要新的 Command 对象？

## OHS 层
- 是否需要新的 HTTP/RPC 端点？
- 是否需要新的 Request/Response DTO？

## assessment.md 输出格式

```markdown
# 层级评估

## 需求概要
（2-3 句话概括需求）

## 评估结论

| 层 | 是否需要 | 原因 |
|----|---------|------|
| Domain | 需要/不需要 | 简述原因 |
| Infrastructure | 需要/不需要 | 简述原因 |
| Application | 需要/不需要 | 简述原因 |
| OHS | 需要/不需要 | 简述原因 |

## 各层概要

### Domain
（需要做什么 / 为什么不需要）

### Infrastructure
（需要做什么 / 为什么不需要）

### Application
（需要做什么 / 为什么不需要）

### OHS
（需要做什么 / 为什么不需要）
```
